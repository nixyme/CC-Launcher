# 项目经验

> 每条原则必须不重复、不冲突、不相互包含。

- Electron 数据文件必须存在 `app.getPath('userData')` 下，严禁写入项目安装目录，避免打包后路径失效。
- 日志系统初始化必须在 `app.whenReady()` 内、`store` 之后立即调用，确保 userData 路径可用再创建日志目录。
- IPC handler 中重复 `require` 同一模块（如 `path`）会造成代码冗余，应统一在文件顶部引入。
- `getUserShellInfo` 等工具函数若被多个模块复用，应提取到公共模块（如 `logger.js` 所在层级），避免在 scheduler 和 ipc-handlers 各自维护一份。
- 定时任务的 shell 环境初始化逻辑与静默执行完全一致，共享同一函数可确保行为一致性。
