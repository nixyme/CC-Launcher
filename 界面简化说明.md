# 界面简化更新说明

## 更新内容

根据你的反馈,我已经简化了界面,移除了多余的按钮。

### 移除的功能

❌ **移除的按钮:**
- `+ 添加命令` - 直接在文本框编辑即可
- `保存命令列表` - 自动保存,无需手动点击
- `📋 执行选中命令` - 简化为只执行所有命令
- `重置命令` - 直接编辑即可

### 保留的功能

✅ **保留的按钮:**
- `▶ 执行` - 执行命令列表中的所有命令
- `📂 查看结果` - 打开结果文件夹

### 新界面布局

```
┌─────────────────────────────────────┐
│ 命令列表                             │
├─────────────────────────────────────┤
│ 每行一条命令,编辑后自动保存          │
│                                     │
│ ┌─────────────────────────────────┐ │
│ │ ./run_script.sh                 │ │
│ │                                 │ │
│ │ ./run.sh                        │ │
│ │                                 │ │
│ │ npm start                       │ │
│ └─────────────────────────────────┘ │
│                                     │
│ [▶ 执行] [📂 查看结果]               │
└─────────────────────────────────────┘
```

## 使用方式

### 添加/删除命令

**之前:**
- 点击"+ 添加命令"按钮
- 点击每条命令旁边的 ❌ 删除

**现在:**
- 直接在文本框中编辑
- 添加新行 = 添加命令
- 删除行 = 删除命令
- 修改文本 = 修改命令

### 保存命令

**之前:**
- 编辑后点击"保存命令列表"

**现在:**
- 自动保存,停止输入1秒后自动保存
- 无需任何操作

### 执行命令

**之前:**
- 两个按钮:"执行所有命令"和"执行选中命令"

**现在:**
- 一个按钮:"▶ 执行"
- 执行所有命令(用 && 连接)

## 优势

1. **更简洁**: 界面更清爽,按钮更少
2. **更直观**: 像编辑普通文本一样编辑命令
3. **更安全**: 自动保存,不会丢失数据
4. **更高效**: 减少点击次数

## 工作流程

```
选择项目 → 编辑命令 → 自动保存 → 点击执行
```

就这么简单!

## 示例

### 编辑命令列表

在文本框中直接输入:

```bash
# 清理缓存
rm -rf .cache

# 构建项目
npm run build

# 运行测试
npm test
```

停止输入1秒后,自动保存 ✅

### 执行命令

点击 `▶ 执行` 按钮,等价于在终端执行:

```bash
rm -rf .cache && npm run build && npm test
```

## 技术细节

- **自动保存**: 使用防抖机制,1秒无输入后自动保存
- **数据存储**: 保存到 `data/projects.json`
- **命令执行**: 所有非空行按顺序用 `&&` 连接执行
