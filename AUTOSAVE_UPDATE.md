# 自动保存和文件展示优化

## 修复内容

### 1. 自动保存功能

在编辑项目时，所有修改会自动保存，无需等到点击"保存"按钮：

**触发自动保存的操作：**
- 命令输入框失焦
- 命令名称输入框失焦
- 命令模式切换（终端/静默）
- 选择可执行文件后
- 删除命令
- URL 输入框失焦
- URL 名称输入框失焦
- 删除 URL
- 文件夹名称输入框失焦
- 选择文件夹后
- 删除文件夹
- 文件名称输入框失焦
- 选择文件后
- 删除文件

**自动保存特性：**
- 防抖机制：500ms 内只保存一次，避免频繁保存
- 轻量级反馈：保存按钮短暂显示"✓ Saved"图标（1秒）
- 仅在编辑模式下生效（新建项目不会自动保存）
- 自动更新项目列表和详情页显示

### 2. 文件展示方式统一

文件输入现在与文件夹输入完全一致：

**修改前：**
- 文件使用文本按钮"Browse"
- 路径输入框 placeholder 为 "Browse file"
- 名称输入框没有 `command-name-field` 样式类
- 删除按钮使用 `btn-remove` 样式

**修改后：**
- 文件使用图标按钮（文件图标 SVG）
- 路径输入框 placeholder 为 "/path/to/file"
- 名称输入框添加 `command-name-field` 样式类
- 删除按钮使用 `btn-remove-command` 样式
- 与文件夹输入完全一致的布局和交互

- `addUrlInputWithValue()` - 添加自动保存监听
- `addFolderInputWithValue()` - 添加自动保存监听

## 用户体验改进

1. **避免信息丢失**：编辑时实时保存，不会因为忘记点击保存而丢失修改
2. **轻量级反馈**：保存时不打断用户操作，只显示短暂的视觉反馈
3. **一致性**：文件和文件夹的操作方式完全一致，降低学习成本
4. **性能优化**：防抖机制避免频繁保存，减少 I/O 操作
